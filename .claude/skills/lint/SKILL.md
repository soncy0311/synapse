# Lint

지식 베이스의 건강 상태를 점검하고 문제를 수정하는 skill이다.

## 사용법

```
/lint
```

## Instructions

### 점검 항목

다음 항목을 순서대로 점검한다:

1. **모순 확인**: 페이지 간 충돌되는 주장이 있는지 확인한다. 발견 시 사용자에게 보고하고 결정을 요청한다 (대체/유지/병기).
2. **stale 노드**: 새로운 소스가 대체한 오래된 주장이 있으면 status를 `stale`로 표시한다.
3. **고아 노드**: 다른 노드에서 링크되지 않는 노드를 발견한다.
   ```bash
   # 각 노드 id에 대해 backlink 확인
   rg "\[\[node-id\]\]" docs/ -l
   ```
4. **누락된 페이지**: 본문에서 `[[node-id]]`로 참조되지만 실제 페이지가 없는 개념을 식별한다.
5. **누락된 교차 참조**: 관련성이 있지만 links가 없는 노드 쌍을 찾아 추가한다.
6. **인덱스 불일치**: LanceDB 인덱스와 실제 docs/ 파일 간 불일치를 수정한다.
   ```bash
   make index
   ```

### 모순 처리 규칙

- Agent는 모순을 발견해도 **절대 독단적으로 기존 내용을 수정하거나 삭제하지 않는다.**
- 사소한 표현 차이는 충돌이 아니다. 핵심 주장이나 사실관계가 다를 때만 사용자에게 보고한다.
- 사용자가 병기를 선택한 경우 양쪽 노드의 "관련 노드" 섹션에 상대 노드를 wikilink로 추가하고, 한 줄 설명에 **모순/충돌 관계임을 명시**한다 (예: `— 본 주장과 모순되는 출처`). frontmatter links에도 상대 노드를 추가해야 한다.

### 결과 보고

점검 완료 후 사용자에게 요약을 보고한다:

- 발견된 문제 수
- 자동 수정한 항목 (교차 참조 추가, 인덱스 재구축 등)
- 사용자 결정이 필요한 항목 (모순, stale 판단 등)
